1. Especificar la Ubicación del Certificado de CA
Una mejor solución es especificar la ubicación de un archivo de certificados de CA (Certificate Authority). Aquí hay un ejemplo de cómo hacerlo:

Descarga el archivo cacert.pem desde curl.haxx.se.

Guárdalo en tu servidor, por ejemplo, en /path/to/cacert.pem.

Configura file_get_contents() para usar este archivo:


$options = array(
    "ssl" => array(
        "cafile" => "/path/to/cacert.pem",
        "verify_peer" => true,
        "verify_peer_name" => true,
    ),
);

$context = stream_context_create($options);
$url = "https://example.com";
$result = file_get_contents($url, false, $context);

if ($result === FALSE) {
    echo "Error fetching URL.";
} else {
    echo $result;
}



2. Verificar la Configuración de PHP
Asegúrate de que tu configuración de PHP esté correcta. Verifica el archivo php.ini para asegurarte de que las configuraciones de OpenSSL estén habilitadas y correctamente configuradas.

openssl.cafile=/path/to/cacert.pem
openssl.capath=/path/to/ca-certs

Reinicia tu servidor web después de hacer cambios en php.ini.


3. Actualizar OpenSSL
Asegúrate de que OpenSSL esté actualizado en tu servidor. En algunos casos, una versión desactualizada de OpenSSL puede causar problemas con la verificación de certificados.

Resumen
La solución más segura y recomendable es usar un archivo de certificados de CA (cacert.pem) para verificar los certificados SSL correctamente. Si estás en un entorno de desarrollo y necesitas ignorar la verificación de SSL temporalmente, puedes configurar file_get_contents() para ignorar las verificaciones de SSL, pero esta no es una solución adecuada para producción.
